# Ansible-Docker-Vagrant (SQLite Project)

## 📌 Описание

Проект демонстрирует сборку и запуск SQLite с использованием различных инструментов:

* C / C++
* CMake
* Docker
* Vagrant
* Ansible
* Conan

---

Этот проект представляет собой полностью автоматизированную инфраструктуру для сборки и запуска C/C++ приложения с использованием SQLite.

Он показывает, как можно:

автоматически поднимать окружение с нуля
собирать проект в Linux и Windows
упаковывать приложение в Docker-контейнер
разворачивать виртуальную машину через Vagrant
управлять установкой зависимостей через Ansible и Conan

Цель проекта — продемонстрировать переносимую и воспроизводимую сборку (reproducible build), где любой пользователь может запустить проект одной командой и получить одинаковый результат независимо от системы.

## Требуемое ПО

Перед запуском необходимо установить:

* VirtualBox
* Vagrant

Остальные инструменты (Ansible, Docker и др.) устанавливаются автоматически.

---

## 🚀 Запуск

```
vagrant up
```

Процесс занимает ~5 минут, так как выполняется полный цикл настройки.

---

## Управление ВМ

```
vagrant halt      # выключить
vagrant destroy -f  # удалить
```

# 🔧 Задание 3 — Сборка и настройка

## 1. Настройка компиляторов

Сборка начиналась в Windows. Стандартных расширений VS Code оказалось недостаточно, поэтому была установлена Visual Studio 2022 для использования компилятора MSVC.

Для сборки под Linux использовалась подсистема WSL, интегрированная в VS Code, что позволило работать в одном редакторе.

Также был изучен и настроен инструмент CMake Tools.

---

## 2. Сборка и решение проблем с путями

Изначально сборка выполнялась как static library, однако это не соответствовало условиям задания, поэтому было принято решение использовать shared library.

После сборки под Linux возникла проблема: исполняемый файл не мог найти библиотеку без ручной настройки путей.

Для решения в CMakeLists.txt был добавлен механизм RPATH:

```
INSTALL_RPATH "$ORIGIN"
BUILD_WITH_INSTALL_RPATH TRUE
```

Это позволило программе автоматически искать `.so` файл в той же директории.

---

## 3. Тестирование

Для проверки работоспособности был создан тестовый `.c` файл.

Программа успешно:

* подключала библиотеку
* выводила версию SQLite

---

## 4. Вывод логов

При сборке на Windows возникли проблемы с кодировкой логов (особенности MSBuild).

Решение:

* перевод логов на английский язык
* использование флага:

```
/v:m
```

Это позволило сделать вывод компактным и похожим на Linux.

# 🐳 Задание 4 — Docker

## Описание

Был создан Dockerfile на базе Alpine Linux для легковесного окружения.

Команда:

```
CMD ["gcc", "--version"]
```

используется для проверки корректной установки компилятора.

---

## Сборка образа

```
docker build -t sqlite-builder .
```

Проверка:

```
docker images
```

---

## Запуск контейнера

```
docker run --rm -v $(pwd):/app sqlite-builder \
gcc -shared -fPIC sqlite3.c -o libsqlite3.so -lpthread -ldl \
2>&1 | tee dockerlog_linux.txt
```

### Объяснение флагов:

* `-shared` — создание динамической библиотеки `.so`
* `-fPIC` — позиционно-независимый код
* `-lpthread` — поддержка многопоточности
* `-ldl` — динамическая загрузка библиотек

### Логи:

* stdout — обычный вывод
* stderr — ошибки
* `tee` — вывод в консоль и файл одновременно

Пустой лог означает успешную сборку.

# ⚡ Как работает Vagrant

## Vagrantfile

* Создает виртуальную машину (Debian)
* Настраивает систему
* Исправляет проблемы с репозиториями и временем
* Устанавливает Ansible

---

## dockerplaybook.yml

Playbook разделен на теги:

### 🔹 install_docker

Установка Docker

### 🔹 Sqlite

Повторяет задания 1–4:

* загрузка SQLite
* сборка через Docker

### 🔹 Conan

Выполнение задания 8

---

### Работа с тегами:

```
ansible-playbook dockerplaybook.yml --tags Sqlite --list-tasks
```
